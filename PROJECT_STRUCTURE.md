## Data Flow Architecture

```
Component Mount
    ↓
Check useAuth() for user & authentication status
    ↓
If not authenticated → Show Login
If authenticated → Show AppContent
    ↓
Page component mounts
    ↓
useEffect hook triggers
    ↓
Fetch data from Supabase
    ↓
Display loading state
    ↓
Render data in table/cards
    ↓
User interaction (click button/submit form)
    ↓
Call Supabase mutation (insert/update/delete)
    ↓
Show loading on button
    ↓
Handle success/error
    ↓
Refetch data from Supabase
    ↓
Update UI with fresh data
```

## Key Files Explained

### Authentication Files

**AuthContext.tsx**
- Manages login/logout logic
- Stores user session in localStorage
- Provides useAuth() hook for components
- Handles both admin and customer logins
- Role-based access control

**Login.tsx**
- Demo credentials display
- Email/password form
- Error handling
- Redirects to dashboard on success

### Layout Files

**Sidebar.tsx**
- Navigation menu with icons
- Role-based menu items (Admin vs Customer)
- Contact Support button
- WhatsApp link
- Logout button
- Collapses on mobile

**Header.tsx**
- Current page title
- Current date display
- Notification bell
- User profile avatar
- Admin user info

### Page Components

Each page component:
- Fetches data from Supabase on mount
- Shows loading state while fetching
- Renders data in table or cards
- Handles CRUD operations
- Provides user feedback (modals, notifications)

**Pattern used in all pages:**
```typescript
useEffect(() => {
  fetchData();
}, []);

const fetchData = async () => {
  try {
    const { data, error } = await supabase
      .from('table')
      .select('*');
    if (error) throw error;
    setData(data);
  } catch (error) {
    console.error('Error:', error);
  } finally {
    setLoading(false);
  }
};
```

### Reusable Components

**Button.tsx**
- Props: variant, size, disabled, onClick
- Variants: primary (blue), secondary (gray), danger (red)
- Sizes: sm, md, lg
- States: normal, hover, disabled, loading

**Table.tsx**
- Props: columns array, data array
- Supports custom cell rendering
- Hover effects on rows
- Responsive scrolling

**Card.tsx**
- Props: children, className, hover
- Rounded corners, subtle shadow
- Optional hover elevation effect

**Input.tsx & Select.tsx**
- Props: label, error, placeholder, value, onChange
- Focused ring state (blue)
- Error state styling
- Optional label display

**Modal.tsx**
- Props: isOpen, onClose, title, children
- Click outside to close (overlay)
- Scrollable content area
- Close button (X icon)

### Support Components

**ContactModal.tsx**
- Displays support contact info
- Phone, email, business hours
- "Call Now" button (tel: link)
- "WhatsApp" button (wa.me link)

**FloatingWhatsAppButton.tsx**
- Fixed position bottom-right
- Green WhatsApp color
- Opens wa.me in new tab
- Visible on all pages
- Doesn't block content

## Type System

**types/index.ts** defines:
- User interface (id, email, role, created_at)
- Apartment interface (id, name, blocks, address, status, created_at)
- Customer interface (id, name, phone, subscription data)
- Subscription interface (id, customer_id, milk_type, qty, price, is_active)
- Delivery interface (id, customer_id, date, quantity, status)
- BillingRecord interface (date, quantity, status)

All components are fully typed with TypeScript.

## Database Connection

**lib/supabase.ts**
- Initializes Supabase client
- Uses VITE_SUPABASE_URL from env
- Uses VITE_SUPABASE_ANON_KEY from env
- Exported for use in all components

Example usage:
```typescript
import { supabase } from '../lib/supabase';

const { data, error } = await supabase
  .from('apartments')
  .select('*')
  .order('created_at', { ascending: false });
```

## Styling Approach

- **Framework:** Tailwind CSS
- **Colors:** Blue (#2563eb) primary, gray neutrals, green for success, red for danger
- **Spacing:** 8px base unit (p-4 = 16px, etc.)
- **Rounded:** 8px corners on most elements
- **Shadows:** Subtle shadows for depth
- **Responsive:** Mobile-first, breakpoints at 768px and 1024px
